3. Контент

Описание модуля: (скришот)

Интерфейс административного раздела системы Incom CMS 2.0 логически разделен на области, обеспечивающие доступ к функциональным возможностям системы. Интерфейс системы построен по иерархическому принципу: вышестоящие области интерфейса переключают зависимые области в тот или иной режим работы. Это позволяет скрыть функциональные возможности системы, в которых нет потребности в данный момент, и вынести на первый план требуемые функции.

В левой части интерфейса CMS расположено меню модулей сайта. С помощью этого меню (навигации) можно выбрать обособленный набор функций, относящийся к определённой функциональной части системы. В зависимости от того, установлены ли те или иные модули, набор функций может различаться.

Набор функций «Контент» содержит инструменты для управления структурой сайта, информационными блоками, веб-формами, опросником, фотогалереей, интернет-магазином.

Набор функций «Настройки» позволяет управлять списками и группами пользователей, настройками продукта (шаблонами сайта, языковыми версиями, обработками адресов, обработкой логов).

 

Модуль сайта - это самостоятельный программно-функциональный элемент сайта, позволяющий реализовать определённые задачи, связанные с функциональными характеристиками элементов сайта.

Раздел сайта - это набор страниц сайта (одной и более), созданный при помощи определённого модуля и имеющий функции, соответствующие тому модулю, в котором он создан. Раздел сайта можно сравнить с “папкой” (директорией), в которую вложены другие разделы, страницы сайта, изображения и прочее. CMS позволяет создавать произвольное количество разделов и подразделов сайта.

Страница сайта - это минимальная информационная единица сайта, имеющая определённую функцию, соответствующую модулю.

3.1. Менеджер файлов:

Это основная рабочая область, в которой происходит большинство операций по управлению сайтом: просмотр элементов, составляющих контент сайта, добавление контента, создание и удаление папок и файлов; настройка меню (навигации) и прочее.

В данном разделе в алфавитном порядке располагаются все файлы и папки (директории), которые находятся непосредственно на сервере.

(Наименование файла соответствует наименованию ссылки, появляющейся в строке состояния при поднесении курсора мыши к меню или наименованию в адресной строке при открытии страницы):

Функция «Переименовать/Удалить»

Данные ссылки позволяют переименовывать или удалять выбранные файлы или директории.

Действие

Данная функция предоставляет возможность совершать доступные операции над файлами и папками: копировать или переносить выбранные папки или файлы в различные директории путём выбора пути (с помощью нажатия кнопки «…») или полного описания пути в поле ввода и нажатия на кнопку «ок».

HTML редактор. Создание и редактирование файла:

Специальный программный модуль реализует HTML-редактор, позволяющий изменять текст и графику веб-страницы в режиме реального времени через браузер. При редактировании выполняется изменение только внутренней (рабочей) области страницы, что позволяет сохранить общий дизайн сайта.

Для редактирования текстового содержания сайта необходимо выбрать из списка файлов необходимую страницу и «кликнуть» дважды на строке с наименованием файла или воспользоваться контекстным меню «Действия», которое отображается при наведении курсора мыши на значок «».

Действия для папок: «открыть», «переименовать», «удалить».

Действия для файлов: «редактировать как html», «редактировать как php», «переименовать», «удалить».

При работе со страницей в режиме «Редактировать как html» открывается область Визуального редактора с панелью, напоминающей MS Word, для удобства работы с содержимым страницы.). Визуальный редактор позволяет редактировать документ в режиме встроенного визуального редактора HTML и форматировать текст аналогично Microsoft Word.

Режим «Редактировать как php» позволяет редактировать исходный код файла. Предназначен для использования web-разработчиками.

Для перехода из одной области редактирования в другую необходимо кликнуть на ссылки (редактировать как html или редактировать как php).

Возможно прямое копирование текста из MS Word, HTML и др. Рекомендуется вставлять текст при помощи функции «Вставить только текст».

В противном случае возможно создание стилей текст (черный цвет текста, ссылки синего цвета и др. детали, выбивающиеся из дизайна сайта).

Добавление фото

При добавлении графического элемента на страницу, необходимо установить курсор на начало/середину/конец строки и выбрать функцию «Вставить/редактировать изображение».

При размещении фото в тексте необходимо заполнить поля «Вертикальный и горизонтальный отступы». Рекомендуется 5-10 пикселей.

В этом же диалоговом окне можно воспользоваться функциями «Ссылка» и «Закачать»:

Окно «Просмотреть на сервере»

При клике на название файла происходит переход на окно «Свойства изображения», и если файл добавляется впервые, его необходимо отправить на сервер с жесткого диска через функцию «Upload» («Загрузить»). Если фото уже было закачано, то его можно найти на сервере через функцию «Просмотреть на сервере».

Внимание! Все отправляемые на сервер файлы и фотографии должны иметь название, состоящее из прописных латинских букв, не разрешено использование пробелов и спецсимволов (@#$%^&*=).

Фотографию можно выровнять по положению на странице (по левому, правому краю, по центру, верху и т. д.). Для изменения размера фото рекомендуем воспользоваться программой Adobe Photoshop.

Для того, чтобы сделать отступ фото от текста, используйте поля «горизонтальный и вертикальный отступы» (размеры указываются в пикселях). Бордюр – черная рамка фото в пикселях (в случае использования рекомендуется не более 1-2 пикселей). При создании ссылки с фотографии бордюр выходит автоматически – синяя рамка (в этом случае необходимо в поле «бордюр» поставить «0»). Окно содержит «предпросмотр», если фото здесь не отображается, то оно не будет отображаться и на странице.

Внимание! Перед отправкой фотографии на сервер рекомендуется оптимизировать фотографию «под web» в прикладных графических программах, что позволяет уменьшить вес фотографии в kb для ее более быстрой загрузки при открытии страницы. Рекомендуемый формат .jpg , .jpeg, .gif.

Создание ссылок

Ссылка – это текстовой фрагмент/графический элемент, ссылающийся на определённый раздел сайта, на адрес электронной почты, на слово в тексте или на какой-либо файл, выложенный в Сети. При клике на этот фрагмент, выделенный, как правило, либо цветом, либо начертанием шрифта, посетитель сайта перемещается в ту область, на которую поставлена ссылка. Т. е. осуществляется переход по ссылке.

Выделяя текст, который будет являться ссылкой, кликаем на значок «Вставить ссылку». Открывается «Диалоговое окно веб-страницы».

В поле «Тип ссылки» существует три вида ссылок:

Ссылка «URL» - предназначена для создания ссылок на прикрепленные документы, фотографии, на другие сайты и на разделы данного сайта.

В поле «URL» необходимо вписать название сайта или название ссылки. Для создания ссылки внутри сайта доменное имя указывать не обязательно. Достаточно указать ссылку со стоящим перед ссылкой слешем (/ru/about.php).

Если необходимо создать ссылку на прикрепленный файл, воспользуйтесь функцией «Просмотреть на сервере» и выберите файл с жесткого диска.

Редактирование меню:

Использование меню обеспечивает удобство навигации по разделам и страницам сайта. Система Incom CMS 2.0 позволяет полностью управлять меню (навигацией) веб-сайта: добавлять новые пункты меню, ссылающиеся на страницы и разделы сайта; изменять названия и ссылки пунктов меню; удалять пункты меню.

Для управления меню и подменю сайта необходимо кликнуть по ссылке «Верхнее меню/Левое меню».

В открывшемся окне появится форма редактирования меню:

• выделить - поле для использования дополнительных возможностей на усмотрение web-мастера. Создание условия, при выполнении которого будет отображаться создаваемый пункт меню;

• - при клике открываются дополнительные подразделы выбранного раздела;

• название - наименование данного раздела, которое отображается в публичной части (возможен ввод латиницы и кириллицы);

• ссылка (ссылка или путь к файлу). Прописывается строго латинскими буквами и отображается в качестве адреса в строке браузера. При создании ссылки обязательно указать расширение .php;

• сортировка - порядковый номер для последовательного отображения пункта меню;

• удалить - удаляет как выбранный пункт, так и подпункты из базы данных.

Функция «Сохранить/Применить»

Сохранить – при выборе данной функции выполняется сохранение страницы и осуществляется переход на предыдущую страницу.

Применить - при выборе данной функции выполняется сохранение изменений. Возможно продолжение редактирования страницы. 

Создание новой папки

Папки создаются для упорядочивания файлов на сервере.

Для создания новой папки (директории) служит пункт меню модулей "Структура сайта", в котором находятся все директории сайта. В папке «ru» находится все текстовая информация по данной (русской) языковой версии.

Для создания новой папки (директории) на сервере необходимо кликнуть на раздел «новая папка». Откроется форма создания новой папки:

• Наименование директории необходимо вводить строго латинскими буквами, исключая спецсимволы. Название папки будет отображаться в качестве адреса в строке браузера.

• Создать индексную страницу раздела - возможность создания файла в новой директории. При открытии нового раздела сайта в браузере по умолчанию первой будет показана эта страница.

• Переход к редактированию страницы – возможность редактирования документа в режиме встроенного визуального редактора HTML. Форматирование текста аналогично Microsoft Word.

Новая папка создаётся в той папке, которая открыта в настоящее время в менеджере. Например, если у вас открыта папка ru/, то новая папка будет добавлена в папку (директорию) /ru/.

Создание файла

Данная форма служит для создания и редактирования файлов. Для создания новой страницы сайта в структуре сайта необходимо кликнуть на раздел «новый файл». (Вновь созданный файл будет расположен либо в корневой папке сайта, либо в той папке сайта, которая будет указана в поле ввода.)

В открывшемся окне появятся поля:

• наименование страницы

Вводится строго латинскими буквами, исключая спецсимволы. Название страницы будет отображаться в качестве ее адреса в адресной строке браузера. При создании файла расширение .php создается автоматически. Дополнительно можно создать файлы с расширением:

.shtml - HTML файл, содержащий некоторые дополнительные действия, выполняемые сервером,

.stm - сокращение для .shtml,

.htm, .html - гипертекстовый документ , Web- страница,

.txt - файл в текстовом виде,

• свойства страницы

Данная секция предназначена для задания и редактирования значений свойств каждой страницы, а также для управления значениями мета-тегов специально для поисковых систем: заголовком (title), ключевыми словами (keywords), описанием (description) страницы. Содержит в своей структуре дополнительные поля для более подробного описания страницы.

• применить шаблон

Созданная страница открывается в дизайне сайта.

Создание нового файла (загрузка)

Вы можете загрузить на сайт любой файл с локального компьютера. Файл будет загружен в папку (директорию), которая открыта в текущий момент в менеджере файлов. Для загрузки файлов различных форматов в директории необходимо кликнуть по ссылке «Закачать файлы». В открывшемся окне появятся поля для выбора файлов с локального компьютера – при выборе файла, его наименование автоматически переносится в поле «имя файла», где можно его переименовать:

Количество загружаемых файлов не ограничено. Максимальный размер одного загружаемого файла ограничивается пропускной способностью сервера (как правило, он составляет 3 Mb). Загрузка файлов большего размера осуществляется через FTP-доступ.

• тип поля - позволяет сгенерировать поле вывода:

Тип поля «Текстовое (text)» - предназначен для ввода текстовой информации.

Тип поля «Текстовое цена (числовое)» - предназначен для вывода стоимости.

Тип поля «Много текста (textarea)» - предназначен для ввода блока текстовой информации.

Тип поля «Выбор (checkbox)» - предназначен для выбора из нескольких заранее определенных позиций.

Тип поля «Список (select)» - предназначен для выбора одного значения из фиксированного выпадающего списка значений.

Тип поля «Список (radio button)» - предназначен для выбора одного из значений. При этом все остальные поля данного типа автоматически станут невыбранными.

Тип поля «html редактор» - предназначен для вывода Визуального редактора.

Тип поля «Рисунок (file)» - предназначен для размещения рисунков.

Тип поля «Файл (file)» - предназначен для размещения прикрепленных файлов.

В зависимости от выбора типа поля добавляется еще некоторое количество полей, таких, как длина поля, формат даты, длина, высота, элементы выпадающего списка, размер, формат.

На сайте пользователи смогут просмотреть информацию о ходе голосования, увидеть результаты ранее проголосовавших пользователей в архиве голосований.

Создание и редактирования опроса.

Эта форма предназначена для редактирования параметров опроса:

Активен/не активен - параметры активности опроса.

ID сорт. – порядковый номер проводимого опроса.

Наименование – заголовок опроса.

Не голосовать дважды (в одной сессии/ с одного IP-адреса) - этот параметр позволяет проголосовать только единожды с одного IP-адреса.

в одной сессии - посетитель не может дважды проголосовать по данному опросу в одной сессии. Выбор данного параметра предпочтительнее, так как каждому посетителю в момент голосования выдается свой уникальный номер, который записывается в cookie посетителя.

с одного IP – данная опция не позволит проголосовать посетителю дважды с одного IP-адреса (необходимо учесть, что если посетитель подключается к Internet через модем (dial-up), то при каждом новом подключении ему выдается новый динамический IP адрес).
